# OLAP Frontend - Angular Приложение

## Описание

Фронтенд приложение для системы OLAP, реализованное на Angular с использованием standalone компонентов и современных практик разработки.

## Структура проекта

```
src/app/
├── core/                    # Основные сервисы (singleton)
│   ├── api/                # API сервисы
│   │   ├── models/         # TypeScript интерфейсы для API
│   │   ├── api-client.service.ts
│   │   └── auth.service.ts
│   ├── interceptors/       # HTTP interceptors
│   └── guards/             # Route guards
├── features/               # Модули функциональности
│   ├── auth/
│   │   ├── login/         # Страница входа
│   │   └── register/      # Страница регистрации
│   └── dashboard/         # Главная страница
├── shared/                # Переиспользуемые компоненты
│   ├── components/
│   ├── pipes/
│   └── directives/
└── layout/                # Shell компоненты (header, footer, etc.)
```

## Технологии

- **Angular 20.3.5** (standalone компоненты)
- **TypeScript** (строгая типизация)
- **RxJS** (реактивное программирование)
- **Reactive Forms** (управление формами)
- **Angular Signals** (реактивные состояния)
- **OnPush Change Detection** (оптимизация производительности)

## Запуск приложения

### Установка зависимостей

```bash
cd olap-frontend
npm install
```

### Запуск dev сервера

```bash
npm start
```

Приложение будет доступно по адресу: `http://localhost:4200`

### Сборка для production

```bash
npm run build
```

## Реализованные страницы

### 1. Страница входа (`/login`)

**Функционал:**
- Ввод номера телефона с маской +7
- Ввод пароля с возможностью показа/скрытия
- Валидация полей
- Обработка ошибок от API:
  - 502 - Серверная ошибка
  - 404 - Пользователь не найден
  - 403 - Неверный пароль
- При успешном входе токен сохраняется в localStorage
- Перенаправление на `/dashboard` после авторизации

### 2. Страница регистрации (`/reg`)

**Функционал:**
- Поле "Имя"
- Поле "Номер телефона" с маской +7 999 999 99 99
- Поле "Email" с валидацией
- Поле "Пароль" с показом/скрытием
- Обработка ошибок:
  - 502 - Серверная ошибка
  - 422 - Неверный формат email
  - 409 - Email или телефон уже зарегистрированы
- При успешной регистрации показ сообщения и редирект на `/login` через 2 секунды

### 3. Dashboard (`/dashboard`)

Заглушка для авторизованных пользователей.

## API Endpoints

### Авторизация

**POST** `/api/auth/login`
```json
{
  "phone": "+79991234567",
  "password": "string"
}
```

**Response 200:**
```json
{
  "session_token": "string"
}
```

### Регистрация

**POST** `/api/auth/register`
```json
{
  "phone": "+79991234567",
  "password": "string",
  "name": "string",
  "email": "user@example.com"
}
```

**Response 201:**
```json
{
  "result": "Пользователь создан"
}
```

## Архитектурные решения

1. **Standalone компоненты** - все компоненты независимые, без NgModules
2. **OnPush стратегия** - оптимизация change detection на всех компонентах
3. **Signals** - использование Angular Signals для реактивных состояний
4. **Feature-based архитектура** - разделение по функциональности, а не по типу файлов
5. **Строгая типизация** - никаких `any`, все API типизированы через интерфейсы
6. **Сервисы для API** - вся логика работы с API инкапсулирована в сервисах
